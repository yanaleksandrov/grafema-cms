Progress bars

Progress bars are used to provide feedback on an action status and inform users of the current progress. Although seemingly small interface elements, they are extremely helpful in managing users' expectations and preventing them from abandoning a process they have initiated.

Simple static progress

To create a default progress bar, add a .progress class to a <progress> element. Thanks to that, you will be able to notify users how long they have to wait for a process to complete.

The current and maximum values are set using the attributes value and max.

Progress bar-loader

You can create a progress bar which shows indeterminate progress by adding .progress-loader to the .progress element.

Loadable progress bars

The animation of the progress bar occurs only when the element hits the viewing area of the screen. The following examples show 2 progress bars. In the first case, the indicator increases from 0 to 100 in 2 seconds. In the second - in the range from 25 to 75 in 1 seconds.
